Introdução ao Backend e Estrutura de um Projeto

O que é backend: Diferenças entre frontend e backend

Backend refere-se à parte "invisível" de uma aplicação, onde são processados os dados, regras de negócios e acessos a bancos de dados. Ao contrário do frontend, que é a interface com a qual o usuário interage, o backend lida com a lógica por trás dessa interação, como autenticação, processamento de pagamentos e manipulação de dados.

Diferenças principais:

  • Frontend: Responsável pela interface do usuário (UI) e pela experiência do usuário (UX).
  • Backend: Responsável por lógica de negócios, manipulação de dados e resposta a requisições do frontend.

Estrutura de um servidor: O que é um servidor web, como ele recebe e responde a requisições

Um servidor web é um software que aceita requisições HTTP de clientes (navegadores, apps) e responde a elas com dados, normalmente em formato HTML, JSON ou XML.

Ele trabalha em um modelo request-response, onde:

  1. O cliente (como um navegador) faz uma requisição (GET, POST, etc.).
  2. O servidor processa essa requisição, muitas vezes consultando bancos de dados, e retorna uma resposta com dados ou uma página web.

Servidores web populares incluem Apache, Nginx, e no caso do NestJS, o servidor embutido utiliza Express ou Fastify.

REST vs. GraphQL: Diferentes arquiteturas para APIs

REST (Representational State Transfer) é uma arquitetura amplamente usada para criar APIs que seguem um padrão de métodos HTTP (GET, POST, PUT, DELETE) para operar sobre recursos.

Cada endpoint na API REST representa um recurso, como /users, /products.

É simples e escalável, mas pode resultar em over-fetching ou under-fetching de dados.

GraphQL é uma alternativa ao REST, onde o cliente pode especificar exatamente quais dados precisa em uma única requisição.

Oferece mais flexibilidade ao evitar múltiplas chamadas à API, permitindo que o cliente solicite exatamente o que precisa em uma consulta.

Introdução ao NestJS: O que é e por que usar

NestJS é um framework para desenvolvimento de aplicações Node.js que usa uma arquitetura modular e organizada, inspirada em práticas de desenvolvimento do Angular. Ele facilita a criação de APIs escaláveis e testáveis.

Por que usar NestJS:

  • Arquitetura modular: Facilita a separação de responsabilidades.
  • Injeção de dependência: Oferece uma forma robusta de gerenciar dependências entre classes.
  • Compatibilidade com outros pacotes: Usa o poder do ecossistema Node.js, integrando-se bem com ferramentas populares como Express, TypeORM, Mongoose, etc.
  • Estrutura consistente: Oferece uma estrutura pré-definida que ajuda a organizar o código.

Instalação e configuração do NestJS: Criando o primeiro projeto

  1. Primeiro instale o controlador de versão do Node: coreybutler/nvm-windows.
  2. Depois de instalado, vá no PowerShell e digite: nvm --version para validar se foi instalado corretamente.
  3. No terminal, digite nvm install lts para instalar a última versão estável do Node.
  4. Com nvm list você verá a lista de versões disponíveis, e com nvm use escolha a versão que foi baixada.
  5. Com o Node instalado na versão LTS, siga os passos de instalação do NestJS: First steps | NestJS - A progressive Node.js framework.

Desafio

Siga os passos de instalação e execute o serviço (apenas o que estiver pré-instalado, sem criação de códigos neste momento). Navegue pelos arquivos e familiarize-se com a estrutura e nomenclatura.

A ideia inicial do desafio é estimular a obtenção de informações estruturadas. Literalmente seguir os passos e desenvolver a zona de aprendizagem proximal.

Pesquise e aprofunde-se sempre que encontrar algo que não entende completamente.
